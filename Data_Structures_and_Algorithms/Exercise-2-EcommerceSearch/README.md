# Exercise 2 - E-commerce Platform Search Function

## Scenario
This exercise implements search functionality for an e-commerce platform.

## Concepts Used
- Big O Notation
- Linear Search
- Binary Search
- Arrays
- Sorting using Comparator

## Big O Notation
Big O notation is used to describe how the running time of an algorithm grows as the input size increases.

## Linear Search
Linear search checks each product one by one.

### Time Complexity
- Best Case: O(1)
- Average Case: O(n)
- Worst Case: O(n)

## Binary Search
Binary search works on sorted data and repeatedly divides the search space into half.

### Time Complexity
- Best Case: O(1)
- Average Case: O(log n)
- Worst Case: O(log n)

## Which is better?
Binary search is better for large sorted data because it has O(log n) complexity.
However, if the data is unsorted and small, linear search is simple and acceptable.
